2 Core Agent Loop

Every skill run follows this lifecycle — from trigger to completion

flowchart TD A[/"🕐 n8n Trigger\n(schedule or webhook)"/] --> B["Create Asana Task\nin Queued section"] B --> C{"Agent Picks Up\nHighest Priority Task"} C --> D["Move Task → Running\nUpdate status"] D --> E["Read Task Description\n+ Custom Fields\nfor Parameters"] E --> F["Execute Skill\n(skill-runner.py → Claude)"] F --> G{Success?} G -->|"✅ Yes"| H{"Has tag:\nneeds-human-review?"} G -->|"❌ No"| I["Move Task → Inbox\n+ Add 'error' tag\n+ Comment with error log"] I --> J[/"👤 Human Investigates\nRetry or Reassign"/] H -->|"Yes — writes data"| K["Post Results as Comment\nMove Task → Needs Review"] H -->|"No — read-only"| L["Post Results as Comment\nMove Task → Done ✅"] K --> M[/"👤 Human Reviews Results"/] M --> N{Approved?} N -->|"✅ Approved"| O["Move Task → Approved\nDependent tasks unblock"] N -->|"❌ Changes needed"| P["Comment with feedback\nMove Task → Queued\n(agent re-runs)"] O --> Q["Move Task → Done ✅"]

8 Review Gate Policy

Only write operations require human approval — read-only skills auto-complete

Skill Type Examples Review? Rationale
Read-only analysis market-intel, PPC health, TACoS, portfolio summary auto-complete No risk — informational only
Competitor research niche-analysis, review-analyzer, competitor-tracker auto-complete No writes, pure research
Bid changes bid-recommender needs review Money at stake
Keyword negation search-term-harvester needs review Can kill good traffic
Campaign creation ppc-campaign-creator needs review Structural + budget commitment
Listing copy listing-manager, listing-optimizer needs review Customer-facing, brand impact
Image planning image-planner needs review Drives photography spend

9 Implementation Phases

Phase 1: Foundation

  • Create Asana project + 6 sections
  • Set up custom fields
  • Create tags (review, auto, anomaly, error)
  • Build Asana helpers in skill-runner.py

Phase 2: Daily Skills

  • n8n creates Asana tasks for daily runs
  • skill-runner reads from Asana queue
  • Auto-complete for routine, alerts for anomalies
  • Slack notifications stay parallel

Phase 3: Dependency Chains

  • Tuesday PPC pipeline as linked tasks
  • Review gates on bid + negation skills
  • n8n polls Approved section for next trigger

Phase 4: Reactive Workflows

  • Anomaly-driven follow-up tasks
  • Product launch templates
  • Listing optimization cycles
  • Scheduled impact measurement
